# zonelease v1.0.0 版本更新日志

**发布日期**: 2026-06-29  
**提交数量**: 7 次提交

---

## 📋 版本概述

v1.0.0 是 zonelease 的首个正式版本，提供面向 Windows DNS / DHCP 的统一管理控制台、Go 后端服务、DNS / DHCP Agent、刷新同步链路、操作审计、通知中心、Docker Compose 部署和 GitHub Actions 发布流水线。

---

## ✨ 新增特性

- 新增 React Start 前端控制台，覆盖仪表板、DNS 管理、DHCP 管理、操作审计和系统设置。
- 新增 Go 后端 API 服务，提供认证、资源管理、刷新同步、通知、审计和 Swagger 文档。
- 新增 DNS Agent 与 DHCP Agent，支持 Windows DNS / DHCP 数据采集与资源操作。
- 新增 DHCP 作用域、租约、保留地址、排除范围、租期和网关等管理能力。
- 新增 DNS 区域和记录管理能力。
- 新增运行态刷新同步、Redis/SSE 事件通知和局部刷新链路。
- 新增 DHCP 作用域导出和前端导出权限控制。
- 新增 Docker Compose 快速部署，单镜像内集成后端、Nginx 和前端 Nitro SSR 服务。
- 新增 GitHub Actions 自动构建、Docker 镜像推送和 Release 产物发布流程。

---

## 🐛 Bug 修复

- 修复容器前端 SSR 运行时缺少依赖导致 `h3-v2` 无法解析的问题。
- 修复容器前端首页因 Vite preview 运行上下文不完整导致 404 的问题。
- 修复 DHCP Go Agent 作用域描述中文乱码问题。
- 修复 DHCP Go Agent MAC 字段分隔符展示不一致问题。
- 修复 DHCP 无限租期识别、租期秒级设置、保留地址修改和租约删除等兼容性问题。

---

## 🚀 性能优化

- 优化 DHCP Go Agent 同步链路，新增作用域详情聚合接口，减少单作用域刷新中的 PowerShell 调用次数。
- 优化后端同步服务，减少重复采集和无效刷新。
- 优化 Docker 前端运行方式，运行镜像只复制 Nitro `.output` 产物。

## 🔧 重构优化

- 前端生产服务改为 TanStack React Start + Nitro 输出，使用 `.output/server/index.mjs` 作为生产入口。

- 优化 DHCP Agent PowerShell 命令封装、字段解析和兼容逻辑。
- 优化后端刷新同步、审计日志和通知载荷结构。
- 整理前端 DHCP 表单校验、IPv4 输入、作用域详情和导出组件。

## 🔄 升级说明

这是首个正式版本，建议使用全新数据库初始化：

```bash
cd deploy
cp .env.example .env
docker compose up -d
```

Docker 镜像内前端已改为 Nitro `.output` 生产服务，手动部署前端时请上传 `.output` 目录，并通过以下方式启动：

```bash
HOST=127.0.0.1 PORT=5173 npm run start
```
