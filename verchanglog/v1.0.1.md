## 📋 版本概述

v1.0.1 修复前端开发服务启用 Nitro Vite 插件后，页面访问 `/api` 请求无法通过 Vite 开发服务器代理转发到后端的问题，同时保持生产构建继续输出 Nitro `.output` 产物。

---

## ✨ 新增特性

- 本版本无新增功能。

---

## 🐛 Bug 修复

- 修复执行 `npm run dev` 时 Nitro 插件介入开发请求链，导致 `/api` 请求无法按 `server.proxy` 配置转发到后端的问题。
- 将 Nitro Vite 插件限制为仅在 `vite build` 命令中启用，恢复开发环境的后端代理链路。
- 保持生产构建继续使用 TanStack React Start + Nitro 生成 `.output` 服务端产物。

---

- 本版本无性能相关变更。

## 🔧 重构优化

## 🔄 升级说明

本版本不需要执行数据库迁移或修改环境变量。更新代码后，开发环境可继续按原方式启动：

```bash
cd frontend
npm run dev
```

开发服务器会将 `/api` 请求代理到 `VITE_API_BASE_URL`；未配置时默认使用 `http://127.0.0.1:8080`。生产环境仍需执行 `npm run build` 生成 Nitro `.output` 产物。
